Demo2:
Grammarly Structured (Working Title)
In this demo, we will dig deeper into the RiTa library and dismantle another important aspect of it: the RiTa grammar program. This program enables you to open the sentence, and remake it within its original structure.

So, what will a computer-made Shakespeare's Sonnet sound like?
What can I make after I learn this?
- You can create a dynamic "poem". What if a poem can change its own text, randomly, towards certain style, or triggered by some external seeds?
- You can rewrite a lyric for a song, following its original syllable and rhyme structure. (To achieve a perfect rhyming effect, you might wanna co-use other RiTa features, such as phoneme analysis.)
What do I need:
- Editor: Visual Studio Code, with Live Server extension installed.
- Your web browser: Chrome (most recommended), Safari, or Firefox.
- Language: Javascript.
1.1
What is grammar?
In natural language, grammar are structural rules and contraints that speakers and writers use to produce language.


As human who possess at least one language system, we don't really realize the existence of grammar when we started learning how to speak back in our childhood years. Some theories believe that language users won't realizing the existence of grammars and other linguistic functions until the age of 4~5, when they've reached certain level of cognitive system development.


We're using English grammar in this demo, but every widely-used language has its own syntax system, they can be very different.



Russian language has a more complex 1st conjugation rule on single/plural subjects of the sentence. Source of this Reference

Some languages, especially those who look 0% alike Latin-based languages, can form very different grammar system. For instance, Chinese doesn't have alphabet, It has no singular and plural like English, it does not have tenses in the Western language sense, it has no genders (masculine, feminine or neuter) and nouns and verbs never change, there are no changing endings. The "grammar" is a highly context-based "pattern" rather than a structural chart to be explained. Therefore, to develop computer literature program for these languages can be a very different task, usually means more tricker works, especially under the condition of which those regions can generally lack the resources of technical and academic sponsorships.
1.2
Why grammar is useful for generative literature?
Unlike human beings who can not only understand language as words, but also read the behavior and draw the subtle context and emotions from the conversation, the computer program lacks that aspect of ability.


However, if we break that highly-integrated human activity into some yes-or-no rules, and use quantitive value to represent those blurred style and possibilities, a computer program can "understand" natural language grammars just like a person.

Program Setting-up
A basic set up: html/css/js/assets.

You can run RiTa on Web. RiTa has its javascript library version that you can download, or use the api link in your html header.

<script src="https://unpkg.com/rita"></script>

To get it powered for our demo today, you also need to run p5.js together with RiTa, in your html header.

<script src="https://cdnjs.cloudflare.com/ajax/libs/p5.js/1.4.1/p5.min.js"></script>

Compared to Demo 1, there is one more thing we're going to be included inside the html head: a grammar.js file. The grammars on this page are written as JSON.
