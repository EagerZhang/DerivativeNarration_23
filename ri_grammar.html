Grammarly Structured (Working Title)
In this demo, we will dig deeper into the RiTa library and dismantle another important aspect of it: the RiTa grammar program. This program enables you to open the sentence, and remake it within its original structure.

1.1
What is grammar?
In natural language, grammar are structural rules and contraints that speakers and writers use to produce language.


As human who possess at least one language system, we don't really realize the existence of grammar when we started learning how to speak back in our childhood years. Some theories believe that language users won't realize the existence of grammars and other linguistic functions until the age of 4~5, when they've reached certain level of cognitive system development.


We're using English grammar in this demo, but every widely-used language has its own syntax system, they can be very different.



Russian language has a more complex 1st conjugation rule on single/plural subjects of the sentence. Source of this Reference

Some languages, especially those who look 0% alike Latin-based languages, can form very different grammar system. For instance, Chinese doesn't have alphabet, It has no singular and plural like English, it does not have tenses in the Western language sense, it has no genders (masculine, feminine or neuter) and nouns and verbs never change, there are no changing endings. The "grammar" is a highly context-based "pattern" rather than a structural chart to be explained. Therefore, to develop computer literature program for these languages can be a very different task, usually means more trickier works, especially under the condition of which those regions can generally lack the resources of technical and academic sponsorships.
1.2
Why grammar is useful for generative literature?
Unlike human beings who can not only understand language as words, but also read the behavior and draw the subtle context and emotions from the conversation, computer program lacks that ability of "reading the air".


However, if we break that highly-integrated human activity into some yes-or-no rules, and use quantitive value to represent those blurred style and possibilities, a computer program can "understand" natural language grammars just like a person.

For this program, you will need two seperated javascript file. One is where your major functions live, the other one is a very clean JSON sheet that serves as your "grammar pool".

----- In your major js file -----

grammar = RiTa.grammar(simple_sentence);
- call to run RiTa.grammar method. The (simple_sentence) means that, your seperated jason file should be start with the same var name. (Will unfold this soon a few paragraphs later)

function mouseReleased() {
let result = grammar.expand();
let simple_sentence = result.split("%");
for (let i = 0; i < lines.length; i++) {
lines[i] = simple_sentence[i];
} }
- Per mouse click, you'll expand a new result of this dynamic sentence. Why we have a for circulation here? It's for multiple line displaying, just to make our following steps simple. You can definitely use a one-line syntax, but I'll recommend we leave it like this as a template, and I'll share how to change the line number in the next section.


----- In your grammar JSON file -----

Now let's take a look at the actual grammar part. In the json file, follow this standard stucture to start your big bracket container. Make sure the var name ("(simple_sentence) ") matches the one in the other .js file, so the other file will be able to identify it and pull the data from the sheet:

let simple_sentence = { [your grammar stuff] }

Now, how to start a simple dynamic sentence? Take the basic "subject-verb-object" structure as an example here. We're replacing the "subject" with "subject1 | subject2 | subject3 | ..." here, and use a method $subject to represent the dynamic placeholder. For verb and object -- vice versa. The complete grammar sheet is as below:

let simple_sentence = {
start: "$subject $verb $object.",
subject: "He | She | Tom",
object: "tacos | bread | pizza | donuts",
verb: "eats | ate | likes | hated | can't have"
}
Note that all of my subject are third-person single here, I make sure my verb conjugation matches my subjects under all circumstances.

Be very careful to not create any typos with JSON! Especially: semi-colon, coma, quote marks, etc. Unlike html and css file which can tolerate old and new syntaxes and even small typos, JSON file tends to require you to make it 100% stick to the textbook.

2.2
A Poetic Machine: "Benjamin Button Closing Lines" Rework
The movie The Curious Case of Benjamin Button depicted the life of Benjamin Button, a fictional role that when his mind ages, his physical body age grows reversely. He was born as an old man and died looking like a baby. The movie featured his bizarred life journey and how he witnesses other people's life as his own life rewinds towards an opposite direction.

The closing lines of the movie was a beautiful monologue that serves as a montage of all the characters:

Some people were born to sit by a river.
Some get struck by lightning.
Some have an ear for music.
Some are artists.
Some swim.
Some know buttons.
Some know Shakespeare.
Some are mothers.
And some people - dance.

Now, what if we can make this poem start moving, as it is partly structured, but also we see the beauty of "randomized life stories"?

The major part of the code is generally the same. Here are a few significant changes in your JSON part:

----- In your grammar JSON file -----

start: "$line0 % $line1 % $line2 % $line3 % $line4 % $line5 % $line6 % $line7",

You might already discover that, for RiGrammar object, you can literally make one $method inside another $method, to create more branching possibilities of your poetic machine.

The % represents a line-changing moment, it'll be pulled by the main js file as it'll expand in a correct, once in a line format.

Besides, you can also create a logic element inside one method, such as the syntax here:

'line2': '$first_word $be ($bad_at | $good_at) $verb_ing;',
It means either the method $bad_at or $good_at will be chosen when it hits this part, the ratio of possibility is 50/50.
